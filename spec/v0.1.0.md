# Spec v0.1.0 — Foundation

> Consolidated spec for the 0.1.0 release. Folds in the original per-feature
> specs (former v0.1.0–v0.6.0). Status: **shipped**. Granular history is in git.

## Vision
A local-first, Ollama-optimised ReAct agent for the CLI: think, plan, and
execute on local hardware with no cloud dependency.

## Scope (shipped)

### Agent core
- **ReAct loop** — Thought → Action → Observation, repeated to a step limit;
  native function-calling for capable models with a text-format parser fallback;
  order-independent loop detection and per-tool failure escalation.
- **Built-in agents** — `build` (full access) and `plan` (read-only); the active
  agent scopes which tools are callable.
- **`ask_user`** — a clarification primitive (ask instead of guessing).
- **Sessions & handoff** — a first-class `Session` owns context, agent, model,
  and cwd; `plan` hands its approach to `build` with a compact "already explored"
  summary (Tab in the TUI), without losing context.
- **`guided` process** — `/guided <task>` plans then hands to build.
- **Slim, model-targeted system prompt** — XML-tagged structure, numbered rules.

### Tools
`bash`, `ssh`/`ssh_upload`/`ssh_download` (incl. Wake-on-LAN), `git`, `fetch`,
`read_file`, `write_file`, `edit_file`, `append_file`, `delete_file`,
`download_file`, `glob`, `grep`. Each returns a `ToolResult`, never throws.

### Interfaces
- **Persistent TUI** (built on `tuir`) — scrollable log, token streaming,
  markdown answers, floating picker/info modals, inline `/`-autocomplete, themes,
  in-TUI cancellation; plus a scriptable one-shot `iaa run`.
- **User-defined agents & skills** (markdown), **provider abstraction** (Ollama
  or OpenAI-compatible), a 24k-token **context manager** with eviction, **session
  logging**, and the `iaa` config + commands surface.
