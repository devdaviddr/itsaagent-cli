# Spec v0.8.0 — Project context (AGENTS.md) + context compaction

Two features brainstormed with the owner; decisions recorded below.

## F-1 — Auto-loaded project context (`AGENTS.md`)  [build first]

**Motivation.** A small local model guesses project conventions (build/test
commands, layout, "don't touch X") badly. Auto-loading a per-project context
file grounds it without the user repeating instructions — the single highest
-leverage grounding for code-folder work.

**Behaviour (chosen: nearest, walk up + dynamic on cd).**
- Walk **up** from the session cwd to the **nearest** `AGENTS.md` (stop at the
  filesystem root / home). Not merged with ancestors — nearest wins.
- Because the system prompt is rebuilt every loop (`refreshSystemPrompt`), the
  file is **re-discovered each turn**, so when the agent `cd`s into a project the
  context loads automatically (and unloads when it leaves).
- Injected as a pinned `## Project context (AGENTS.md)` section in the system
  prompt; **size-capped** (~8 KB) with a truncation note so it can't crowd the
  window.
- Config: `projectContext: boolean` (default `true`) to disable.

**Acceptance criteria.**
- [ ] `findProjectContext(dir)` returns the nearest `AGENTS.md` content (walking
      up), `null` when none, and caps oversized files.
- [ ] The system prompt includes the project-context block when an `AGENTS.md`
      is present in/above the session cwd, and omits it otherwise.
- [ ] `cd`-ing into a directory with an `AGENTS.md` loads it on the next turn
      (dynamic); leaving unloads it. Honors `projectContext: false`.
- [ ] Tests cover discovery, capping, prompt inclusion, and the runtime pickup.

**Files:** `src/agent/projectContext.ts` (new), `src/agent/promptBuilder.ts`
(option + section), `src/agent/AgentRuntime.ts` (`refreshSystemPrompt`),
`src/types.ts` + `src/cli/config.ts` (the flag).

## F-2 — Context compaction at a threshold (hybrid)  [build next]

**Motivation.** Today the context only acts at 100% (oldest-first eviction). Act
proactively at ~80% and **compress without losing meaning**.

**Behaviour (chosen: hybrid — structured by default, optional LLM).**
- Trigger in `ContextManager` when `usage().ratio ≥ compactionThreshold` (0.8),
  compacting down toward ~60%. Always keep verbatim: system prompt, original
  task, and the last K turns.
- **Structured (default, no LLM):** dedup superseded file reads (keep newest per
  path), recency-graduated truncation of old tool results (full → first-line →
  folded into the existing deterministic digest).
- **Optional LLM pass** (`compaction: "summarize"`): one local-model call folds
  older turns into a pinned `[CONVERSATION SUMMARY]`. Opt-in (7B quality varies).
- Config: `compaction: "structured" | "summarize" | "off"` (default
  `"structured"`), `compactionThreshold` (default `0.8`).

**Acceptance criteria.**
- [ ] At ≥80% the context is compacted below the threshold without dropping the
      task/system/most-recent turns; verified by tokens-before > tokens-after.
- [ ] Superseded reads deduped; old tool results truncated; digest retained.
- [ ] A long-session e2e scenario still completes after compaction (trajectory
      scored); reliability not worse vs baseline.
- [ ] `compaction: "off"` falls back to current eviction.

**Files:** `src/agent/ContextManager.ts`, `src/agent/Session.ts` (summary hook),
`src/cli/config.ts` + `src/types.ts`, `tests/e2e/e2e.ts` (long-session scenario).
