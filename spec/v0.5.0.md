# ItsAAgent v0.5.0 — Feature Spec

**Branch:** `feat/tui-ux-polish`
**Status:** Implemented — released as v0.5.0
**Target:** v0.5.0 (**minor** — additive/UX; CLI and config contracts preserved)
**Process:** Spec-Driven Development (recorded as-built; this work evolved through tight UX iteration on the persistent TUI introduced in v0.4.0).

---

## Vision

v0.4.0 unified the interactive experience into one persistent Ink TUI. v0.5.0 makes that TUI **genuinely usable and good-looking**: real floating modals, line-by-line scrollback, token streaming, markdown styling, themes you can customise, and a clean borderless layout — while fixing the rough edges (focus, flicker, overflow) that made the first cut feel buggy.

---

## Background

The v0.4.0 Ink TUI rendered well but hit hard limits: stock Ink can't do overlay modals (ink#182), flickered at full height, couldn't scroll within a long message, and didn't stream responses. This release addresses each, including migrating the persistent view layer to the **`tuir`** Ink-fork for real overlays.

---

## Features (as built)

### F-01 · tuir migration — true overlay modals
The persistent TUI moved from Ink to `tuir` (Ink-fork with z-index `Modal` + focus-trapping). The legacy one-shot `iaa run` view stays on Ink (separate render path).
- **Acceptance:** `/agent`, `/model`, `/theme`, `/tools` open centered floating dialogs over the conversation; Esc closes; focus returns to the input.

### F-02 · Line-level scrollback
The transcript is flattened to styled, wrapped lines (`flatten.ts`) and windowed at the **line** level — scroll line-by-line through the whole history, not just between messages.
- **Keys:** ↑/↓ one line, Ctrl+U/Ctrl+D half-page, Esc jumps to latest, **mouse/trackpad wheel** scrolls (mouse reporting on).
- **Acceptance:** a long answer no longer overflows the input; full history is reachable; `windowLines` is pure + tested.

### F-03 · Token streaming
`OllamaProvider` always streams (`stream:true`) and accumulates `tool_calls` across chunks, so responses render token-by-token even with tools attached.
- **Acceptance:** responses appear incrementally; tool calls still execute.

### F-04 · Markdown styling + colored chat area
Answers render fenced code blocks, headings, and prose in distinct theme colors; the transcript sits in its own panel-colored area.
- **Acceptance:** `markdownLines` colors code/headings/prose (tested).

### F-05 · Theming — backgrounds, weight, custom themes
Theme gains optional `background`/`panel` fills and a `bold` weight toggle; new built-ins `dracula`, `nord`, `gruvbox`; a fully user-defined `custom` theme via `config.customTheme`.
- **Acceptance:** `resolveTheme` merges overrides; `/theme` lists built-ins + `custom`.

### F-06 · Browsable `/tools`, consistent modals
Every command is a modal; `/tools` is a selectable list → pick a tool → detail view (full params). `/help`, `/about` are read-only info modals.

### F-07 · Borderless, flicker-free, responsive
- No `<Viewport>` (it forced full-height → tuir's clear-every-frame flicker); a plain root box one row short keeps the diff path.
- Input re-arm effect (`rearmTick`) keeps the keyboard alive after `/clear` and idle-staying commands.
- Borderless input + palette; chat keeps its panel background.
