# Spec v0.2.0 — Reliable dynamic harness + sessions

> Consolidated spec for the 0.2.0 release. Folds in the former v0.7.0
> dynamic-harness spec plus the tool/reliability and sessions work shipped
> alongside it. Status: **shipped** (one item deferred — see below).

## Vision
Turn the static ReAct loop into a *dynamic, self-correcting* harness that plans,
verifies its own work, and completes whole tasks reliably on small local models —
and make every change measurable. Add durable sessions.

## Scope (shipped)

### Correctness & prompt reliability
- **`num_ctx`** — request the full window (input budget + output headroom) so the
  server stops silently truncating the context.
- **`— OK`/`— FAILED` tool framing**, **mid-task `<answer>` re-prompt**, and a
  **worked few-shot exemplar** to curb hallucinated success / premature stops.
- **Native-vs-text prompt branch** — drop the XML `<tool_call>` schema when the
  model uses native function-calling.
- **Per-model sampling profiles** (temperature/numPredict/stop, config-overridable)
  and **bounded provider retries** for cold-start hiccups.

### Self-correction & completion
- **`run_tests` tool** (auto-detects npm/pnpm/pytest/cargo/make).
- **Verification gate** — `build` confirms its work with a tool before finishing
  (fires once per run, only when a mutation ran).
- **Best-effort recovery** turn instead of dead-end aborts.
- **Build plans-first and builds the whole solution; plan gathers info** until it
  can plan.

### Tool fixes
- **`make_directory`** (real folders, not 0-byte files); **string-based
  `edit_file`** (`old_string`/`new_string`, no line miscounts); **literal-`\n`
  repair**; **shared session cwd** across `bash`/`git`/file tools.

### Sessions
- **Persistence & resume** — autosave per turn; `iaa sessions`,
  `iaa chat --resume [id]`; **`/save`** transcript export.

### Measurement
- **Live e2e suite** (`pnpm e2e`) — real runtime vs a live model, **reliability
  runs (`--runs N`)**, **trajectory scoring**, and a **regression baseline
  (`--compare` / `baseline.json`)**.

## Deferred
- **Constrained tool-call decoding** (Ollama `format`/JSON schema) — invasive and
  largely redundant once native tool calls + `num_ctx` are in place.

## Definition of done
- [x] Build + tests green; reliability not worse vs baseline; tagged `v0.2.0`.
