# Spec v0.9.0 ‚Äî Git awareness + codebase repo map

Two features so `iaa` understands the codebase it's opened in. Decisions recorded.

## F-1 ‚Äî Git awareness (status line + agent-aware prompt block)

**Motivation.** Opening `iaa` in a repo, neither the user nor the agent sees the
repo state; the agent must run `git status` to learn anything. Surface it.

**Behaviour.**
- `gitContext(cwd)` (read-only, mirrors `projectContext`): walk up to the repo,
  return branch, changed-file count + names (capped), and the last ~5 commits.
- **Agent-aware prompt block**: pinned `## Git` section injected into the system
  prompt (rebuilt each turn, like AGENTS.md) so the agent knows branch + changed
  files + recent commits without calling git first.
- **TUI status line**: show `‚éá <branch> ¬∑ <n> changed` (or `clean`).
- Config: `gitContext: boolean` (default true). Only injects inside a repo.

**Acceptance.**
- [ ] `gitContext()` returns branch/changed/commits in a repo, `null` outside one.
- [ ] System prompt includes the `## Git` block in a repo, omits it otherwise and
      when `gitContext:false`.
- [ ] TUI status line shows branch + dirty/clean.
- [ ] Tests: discovery in/out of a repo; prompt inclusion; runtime pickup.

**Files:** `src/agent/gitContext.ts` (new), `promptBuilder.ts` (option+section),
`AgentRuntime.ts` (`refreshSystemPrompt`), `types.ts`+`config.ts` (flag), TUI
status line.

## F-2 ‚Äî Structural repo map tool (`repo_map`)

**Motivation.** A 7B navigates a codebase poorly because it doesn't know *where*
to look. A compact, deterministic map of files + their top-level symbols lets it
orient before answering questions or editing ‚Äî so its grep/read lands.

**Behaviour (chosen: structural first; embeddings deferred to F-3).**
- `repo_map` tool: walk the repo (skip node_modules/.git/dist/build/vendor/etc.),
  list code files grouped by directory, and extract top-level symbols per file
  (functions/classes/exports via language-aware regex). Ranked, capped (~6 KB).
  Optional `path` arg to scope to a subdir. Read-only (plan agent may use it).
- The agent calls it to understand structure before answering "how does X work"
  or "where is Y" ‚Äî then greps/reads the named files.

**Acceptance.**
- [ ] `repo_map` returns a file‚Üísymbols map for the session cwd, skipping ignored
      dirs, capped, with a truncation note.
- [ ] Registered in `getDefaultTools()`; not a mutation (plan-permitted).
- [ ] Tests: symbol extraction (js/ts + py), ignored-dir skipping, capping.

**Files:** `src/tools/repoMap.ts` (new), `src/tools/index.ts`.

## F-3 ‚Äî Embeddings / semantic search (DEFERRED)
`iaa index` + a local embedding model (e.g. `nomic-embed-text`) + a `search_code`

store, indexing pipeline, staleness). Spec separately once F-1/F-2 land.
