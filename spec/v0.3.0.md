# Spec v0.3.0 — Codebase & context awareness

> Consolidated spec for the 0.3.0 release. Folds in the former v0.8.0
> (AGENTS.md + compaction) and v0.9.0 (git awareness + repo_map) specs, plus the
> guided plan→build quality fixes. Status: **shipped** (one item deferred).

## Vision
Make the agent *understand the codebase it opens in* and *manage its context
intelligently*, so a small local model stays grounded and effective on real work.

## Scope (shipped)

### Codebase awareness
- **Auto-loaded project context (`AGENTS.md`)** — load the nearest `AGENTS.md`
  (walking up from the session cwd) into the prompt; re-discovered each turn, so it
  loads/unloads dynamically as the agent `cd`s between projects. `projectContext`
  flag, size-capped.
- **Git awareness** — a pinned `## Git` block (branch, changed files, recent
  commits) injected into the prompt and refreshed each turn; TUI status line shows
  `⎇ <branch> · <n> changed`. `gitContext` flag.
- **`repo_map` tool** — a structural index: code files grouped by directory with
  their top-level functions/classes/exports (language-aware, capped, optional
  subdir). Read-only; the agent calls it to orient before answering codebase
  questions. Plan-permitted.

### Context management
- **Compaction at a threshold (hybrid)** — at ≥80% the context is compressed, not
  just evicted: `"structured"` (default, no LLM — cap old tool results, stub
  superseded reads) and opt-in `"summarize"` (fold older turns into one pinned
  summary via a local-model call). `compaction`/`compactionThreshold` config.

### Guided plan→build quality
- **Agent-executable plans** — `plan` writes concrete tool steps (no human/UI
  instructions); the handoff carries out every step; the **verification gate
  enforces completeness** (files + dependency installs + commands); shell paths
  are quoted.
- **`bash cwd` is one-off** — no longer persists, so repeating a relative `cwd`
  stops compounding folders.

## Deferred (→ next spec)
- **Embeddings / semantic code search** — `iaa index` + a local embedding model
  (e.g. `nomic-embed-text`) + a `search_code` retrieval tool. The bigger lever for
  true codebase Q&A on a small model; larger (vector store, indexing, staleness).

## Definition of done
- [x] Build + tests green; verified live; tagged `v0.3.0`.
