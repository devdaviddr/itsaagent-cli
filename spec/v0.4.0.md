# Spec v0.4.0 — Accuracy, reliability & semantic retrieval

> Spec for the 0.4.0 release. Targets the failure modes that make a small local
> model (`qwen2.5-coder-7b` / `mistral`) unreliable on real coding and non-coding
> tasks: looping, blind edits, false-success, lost context, and shallow
> verification. Also adds codebase grounding via local embeddings. The thesis is
> unchanged — *the harness is the product*: tighter feedback loops let a weak
> local engine punch above its weight. Status: **shipped**.

## Vision
Make the agent *accurate* and *self-correcting* on local models — it should
finish what it starts, catch its own mistakes in-loop, manage a tiny context
window well, and verify the right things for the kind of task at hand (code vs
general). Give the agentic coder a way to find relevant code by meaning so it can
work in a codebase that doesn't fit its window.

## Scope (shipped)

### Edit reliability & self-correction
- **Fuzzy/anchored `edit_file`** — when an exact `old_string` match fails (a local
  model rarely reproduces surrounding code verbatim), fall back to a
  whitespace-normalized match, then a line-trim anchored match, replacing the
  correct original span and reporting which strategy matched and at what line.
  Ambiguous matches error rather than guess. Pure `locateEdit()` helper, unit-tested.
- **Inline diagnostics after every edit** — after a successful `write_file`/
  `edit_file`, run the best *locally-available* checker (`tsc --noEmit` / `eslint`
  for TS-JS, `ruff` / `py_compile` for Python) and append the result to the tool
  output, so the model sees real type/lint errors and self-corrects in the same
  loop. Never installs, never blocks, tight timeout, capped output
  (`checkDiagnostics`). `node --check` restricted to `.js/.mjs/.cjs` (it
  false-fails on TS syntax).

### Verification (code vs general)
- **Task-type-aware verification gate** — `detectTaskType()` routes a task to one
  of two paths. **Code:** a system-generated deliverables checklist (every file
  written, dependency install, and detected test suite), and the final `<answer>`
  is rejected unless backed by a real tool call. **General:** a completeness
  self-critique (re-read the request, confirm every part is covered).
- **Escalating completion nudges** — the "that's a status, not a finished task"
  nudge fires up to twice; verify rejections capped.

### Context management
- **Real token accounting** — capture Ollama `prompt_eval_count`/`eval_count` and
  calibrate the estimate (EMA), so compaction fires on actual usage, not a fixed
  chars-per-token guess.
- **Context preservation** — old tool-result payloads capped at 600 chars (was
  200) and `FAILED` results never truncated, so diagnostic context survives
  compaction; tool-result display cap raised to 10k.
- **Step-budget awareness** — nudges at 50/75/90% of the step budget push the
  agent to finish over explore (notice-flagged, injected once each).

### Throughput & prompt economy
- **Parallel tool execution** — multiple read-only native tool calls in one turn
  run concurrently; mutations stay sequential (shared `processToolCall`).
- **Compact prompt mode** — in native tool-calling mode the system prompt drops
  redundant parameter descriptions (the JSON schema already carries them),
  reclaiming ~300–500 tokens/call. On by default for the Ollama provider
  (`compactPrompt`).

### Codebase grounding (semantic retrieval)
- **`iaa index` + `search_code` tool** — retrieve code by *meaning*. `iaa index
  [path]` walks the repo (reusing `repo_map`'s ignore-dir/extension sets), splits
  files into overlapping line-windows, embeds each via a local Ollama embedding
  model (`nomic-embed-text` by default, `/api/embed`), and persists vectors to
  `~/.config/ai-cli/index/<hash>.json`. `search_code` embeds a query, ranks
  chunks by cosine similarity, returns `path:start-end (score)` + snippet.
  Read-only — available to `plan` and (via `tools: all`) `build`. New `embedModel`
  config key. Live search requires `ollama pull nomic-embed-text` + an `iaa index`
  run first. Pure chunk/cosine/rank helpers unit-tested.

### Correctness fixes (folded in)
- **Looping** — tightened recency nudge (3 repeats in the last 6 calls, was 5/8)
  and a directive reprompt when the model narrates an action without taking it.
- **Working directory** — the rules block and project/git context use the live
  session cwd (`getSessionCwd()`), not the process launch dir, so paths are right
  after a `cd`.
- **No duplicate system message** — `initSession()` replaces the system prompt
  atomically.
- **Plan agent git safety** — `checkout`/`reset`/`clean`/`rebase`/`merge`/`push`/
  `pull`/`fetch`/`stash` blocked for the read-only agent; backslash-escaped quotes
  in git args parsed correctly.
- **Handoff seed** — carries the *most recent* commands run (was the first 10).
- **Streaming** — NDJSON decoder flushed at stream end so a split multi-byte
  character isn't dropped.
- **Single version source** — `VERSION` constant drives both the CLI `--version`
  and the TUI status bar (was hardcoded in two places).

## Deferred (→ next spec)
- **Multi-model routing** — per-agent model selection (`agentModels`), multiple
  provider endpoints, capability registry, and fallback chains. Foundations exist
  (`AgentDefinition.model`, `modelProfiles`); config surface not yet built.
- **Tree-sitter symbol extraction** for `repo_map` (regex today).
- **Structured (JSON) plans** with a build-side step tracker.
- **Tool approval / `--safe` mode** (per-tool allow/deny/require-approval).
